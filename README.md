# Расчёт выплат сборщикам и менеджерам

Полноценное приложение для импорта объектов из amoCRM и расчёта выплат сборщикам, менеджерам и сотрудникам с фиксированными окладами.

## Стек

- Frontend: React + TypeScript (Vite)
- Backend: Node.js (Express) + TypeScript
- ORM: Prisma + SQLite
- Тесты бизнес-логики: Jest

## Быстрый старт

```bash
npm install
npm run prisma:generate
npm run prisma:push
npm run prisma:seed
npm run dev
```

Приложение стартует на порту `3000` (backend) и `5173` (frontend с прокси на API). В production-сборке фронтенд раздаётся Express-приложением.

## Скрипты

- `npm run dev` — одновременный запуск клиента (Vite) и сервера (ts-node-dev)
- `npm run build` — сборка shared-библиотеки, backend и фронтенда
- `npm run build:server` / `npm run build:client` / `npm run build:shared` — отдельные стадии сборки
- `npm run test` — Jest-тесты расчётных алгоритмов
- `npm run prisma:generate` — генерация Prisma Client
- `npm run prisma:push` — применение схемы к SQLite
- `npm run prisma:seed` — заполнение базовых справочников (константы, менеджеры, оклады, сборщики и бригады)

## Импорт CSV

1. Загрузите CSV из amoCRM (UTF-8, разделитель `,` или `;`).
2. На шаге сопоставления выберите обязательные поля «Название сделки» и «Бюджет ₽», опционально — ID, Ответственный, даты.
3. Система автоматически очистит значения бюджета, преобразует даты `DD.MM.YYYY HH:MM:SS`, сопоставит менеджеров по алиасам и добавит сделки в базу.

В каталоге `fixtures/` лежит пример файла `amocrm_sample.csv`.

## Настройки

Раздел «Настройки» позволяет управлять:

- Константами (фикс «Нагрузчики», процент фонда сборщиков, режим округления)
- Менеджерами, их процентами и алиасами
- Фиксированными окладами (включая Влад — дизайнер, 40 000 ₽)
- Списком сборщиков
- Бригадами по два человека с долями распределения (по умолчанию 50/50)

Все изменения сохраняются в SQLite через Prisma.

## Расчёт выплат

1. Отфильтруйте сделки по году и типу даты (продажа/монтаж).
2. Выберите нужные объекты, укажите стоимостные корректировки, процент выплаты и бригаду.
3. Расчёт базы: `max(0, бюджет − столешница − доставка − 4 000)`.
4. Фонд сборщиков: `база × 10% × (% выплаты / 100)` с распределением по долям бригады.
5. Менеджеры получают проценты от полного бюджета (продажа — в месяц +1, монтаж — по факту статуса/даты монтажа).
6. Активные оклады автоматически входят в итог, можно временно отключить чекбоксом.

Результаты расчёта агрегируются по сотрудникам, доступны экспорт в CSV/XLSX и печатная форма с подписями.

## Тесты

В `shared/__tests__/calculations.test.ts` покрыты контрольные сценарии из ТЗ:

1. 100% выплата при базе 826 000 ₽ → фонд 82 600 ₽, распределение 50/50.
2. Частичная выплата 50%.
3. Частичная выплата 30% для бригады 60/40.
4. Нулевая база при превышении затрат.
5. Начисления менеджерам (продажа/монтаж).
6. Учёт фиксированных окладов, включая Влад (дизайнер).

Запуск: `npm run test`.

## Структура проекта

```
shared/            — общие типы и расчётные функции
server/            — Express API, импорт CSV, Prisma
frontend/          — React UI (Vite)
prisma/schema.prisma — модель данных SQLite
fixtures/          — пример CSV для импорта
```

## Экспорт и печать

Во вкладке «Ведомость выплат» доступны:

- CSV и XLSX выгрузка по сотрудникам с детализацией
- Печатная форма A4 с итогами и местом для подписей

## Переменные окружения

- `PORT` — порт backend (по умолчанию 3000)
- `DATABASE_URL` — путь к SQLite (по умолчанию `file:./dev.db` из Prisma)

Настройки загружаются из `.env`, если файл присутствует в корне.
